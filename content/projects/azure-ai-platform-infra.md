An internal Azure platform that gives product teams fast, self-service cloud
environments with consistent guardrails and least-privilege isolation — while
keeping a human-reviewed gate and safe teardown/revival on every change.

## What I built

- **Designed the platform skeleton, two-identity authorization model, and remote
  state scheme.** A privileged platform service principal provisions each
  team's resource group, scoped service principal, security group, and RBAC;
  a restricted team-only principal handles day-two workload applies — keeping
  identity and RBAC objects out of the team's reach by design.
- **Built the shared "hub" Terraform stack** of platform singletons — API
  Management, App Service Plan, Key Vault, Storage, Application Insights — plus
  hub networking: Azure Firewall, Application Gateway, private DNS zones, and
  private endpoints for data-plane isolation.
- **Authored the per-team templates** with render-token placeholders the
  onboarding scaffolder renders into per-team stacks, enforcing a hard split
  where one identity cannot manage another's directory/RBAC objects.
- **Implemented manifest-driven team onboarding and decommissioning.** Add a
  team to a JSON manifest (or a key to a decommission manifest) and symmetric
  CI flows render code, open one-team-per-MR, plan, apply — and on
  decommission, destroy workload-then-initial and archive Terraform files for
  safe re-onboarding later.
- **Wrote the CI/lifecycle automation** (onboard, decommission, MR plan-comment
  shell scripts) that the pipelines run for plan/apply orchestration across hub
  and team stacks.
- **Hardened the security posture end-to-end:** disabled storage key-based auth
  and switched the provider to Azure AD for storage data-plane access; secrets
  never touch git — team SPN credentials are generated by the initial stack and
  held in the platform Key Vault.
- **Produced deep architecture documentation and an interactive architecture
  knowledge graph** (~390 nodes / ~380 edges across 140+ files) to make the
  deliberate two-stack split navigable for the wider team.
